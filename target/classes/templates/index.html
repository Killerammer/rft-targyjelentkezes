Tárgy Jelentkező Alkalmazás

Ezaz alkalamzás az RFT feladatokhoz készült.

Elvárás:
Előadó képes tárgyakat hozzáadni a rendszerhez, és ehhez bizonyos adatokat megadni (többek között tárgy neve, időpontja). A hallgatók képesek jelentkezni az órákra. Mindez egy adatbázisban tárolásra kerül.

A csoportfeladat web alkalmazásához:
  • 1. Unit tesztek készítése. Valós teszteket kell írni, azaz nem az órán mutatott "hackelt" teszteket, amelyek valótlan kód lefedettséget mutatnak. A webalkalmazás Service osztályainak a lefedettsége 80% kell, hogy legyen.
  • 2. Git: Tegyétek közzé a projektet egy git repositoryban, majd az oktatók email címére küldjétek el a linket vagy a meghívót ha privát a repository. Emailek: norbert_gulacsi@epam.com, laszlo_szikszai@epam.com, attila_jakubat@epam.com, david_borbely@epam.com, kornel_filep@epam.com. A különböző feladatokhoz a megoldásokat pull requesteken keresztül tegyétek bele az alkalmazásba, azaz alkalmazzatok brancheket.
  • 3. API tesztek készítése. Az összes API methodra egy pozitív teszt eset, ellenőrizzétek a status code-t, és a paramétereket ahol lehet. Beégetett IDkat ne tartalmazzanak. (pl.: ne legyen olyan teszt ami elkéri a 123 IDjű usert fixen, hanem ilyenkor hozzatok létre egy usert az api teszt futtatás előtt kóddal, majd azt kérjétek el)
  • 4. Készítsetek egy CI pipelinet. Itt egy CI pipeline szükséges ami lebuildeli az alkalmazást és lefuttatja az 1. pontban említett unit teszteket (GitHub Actions, GitLab CI vagy Jenkins).
  • 5. E2E tesztek készítése. Pozitív esetekre készítsetek UI teszteket az alap felhasználói funkciókra. Az E2E tesztek lehetnek ugyan abban a repositoryban is mint a fő alkalmazás, de ki is rakhatjátok akár egy külön repoba, csak erről akkor írjatok emailt a fent említett email címekre.
  • 6. Performance teszt: Performance tesztek készítése. 1 JMeter script ami használja az összes API methodot. Itt se legyenek beégetett értékek.
